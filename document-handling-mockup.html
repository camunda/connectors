Document Handling — Connector Properties UI

Property panel states for document source selection (input) and return format (output)

Document Input

Every Document-typed field gets a source dropdown. A hidden FEEL composer field assembles the final JSON and sends it to the connector via zeebe:input. For List<Document> fields, a mode toggle switches between a single-document source dropdown and a multi-document FEEL array field.

Document Output — Return Format

Replaces asFile / asDocument / storeResponse booleans across 6 connectors. ETG detects DocumentResponse type and generates this 3-option dropdown. For base64 encoding, use FEEL's base64encode() on the text result.

Blue FEEL fields — accept FEEL expressions. Value is evaluated at runtime.

Visible fields use zeebe:property binding (stored in BPMN XML, not sent to connector). A hidden FEEL composer field uses zeebe:input — the only binding the connector receives.

Base64 encoding: Not a dropdown option — use FEEL's base64encode(result) after receiving the text result instead.

Breaking change: Old boolean toggles (asFile, asDocument, storeResponse) are removed. New element template version required. Migration guide needed.
