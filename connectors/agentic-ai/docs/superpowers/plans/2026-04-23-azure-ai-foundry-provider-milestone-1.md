# Azure AI Foundry Provider — Milestone 1 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Land the new "Azure AI Foundry" provider option in the AI Agent connector's element template (Task + Sub-process variants) so the form can be demonstrated in the Camunda Modeler ahead of any runtime implementation.

**Architecture:** Add a new annotated Java record (`AzureFoundryProviderConfiguration`) registered as a sealed subtype of `ProviderConfiguration`. Element-template generation reads the annotations and produces JSON templates automatically; bumping the `@ElementTemplate` version triggers `versionHistoryEnabled` to move the previous version to `versioned/`. Runtime dispatch is a stub that throws `UnsupportedOperationException` — runtime implementation lands in Milestone 2 after demo feedback.

**Tech Stack:** Java 21, Maven, Jackson (sealed-type discrimination), `io.camunda.connector:element-template-generator` (annotation-driven JSON generation), gmavenplus-plugin (Groovy transform for the job-worker variant).

**Spec reference:** `connectors/agentic-ai/docs/superpowers/specs/2026-04-23-azure-ai-foundry-provider-design.md`

**Out of scope (Milestone 2):** SDK integration, adapter, factory dispatch wiring, deserialization tests, ArchUnit rules, runtime tests, ADR. Documented in the spec.
